# 回測驗證計畫 v1.0

## 1. 樣本設計

**期間**:2023-07 至 2026-06(約2.5年、每市場約620個交易日),涵蓋三種 regime:2024台股AI多頭、2025年初半導體回檔、2026上半年下修環境——多頭、空頭、盤整齊備,避免單一行情擬合。

**Universe 重建(存活者偏差防治)**:
- 台股:以每季市值快照重建當期符合「100億市值+3,000張日均量」的名單,含已下市/被併購標的
- 美股:Nasdaq 100 歷史成分自 Wikipedia revision history 重建,SOX 成分以歷次調整公告重建
- 嚴禁以2026年今日名單回推

**Point-in-time 對齊鐵律**:13F以filing_date、SI以publication_date、財報以公告日+1個交易日對齊;台股法人資料以T日盤後公布、T+1可用。

## 2. 回測指標組(降階版聲明)

估值層僅用可歷史重建的三項:EV/EBITDA產業分位數、FCF yield、橫斷面P/E分位數。**forward P/E自身歷史Z-score與PEG不進回測**(point-in-time預估資料免費源不可得,回填即lookahead)。因此:

> 回測結果 = 降階版估值層的表現下界。實盤版含forward P/E Z-score的增量貢獻,以walk-forward方式於上線後逐季驗證(有/無此因子的候選清單勝率差異),累積12個月樣本後決定去留。

## 3. 驗證程序

1. **主回測**:FIXED三重門檻,全期間、依trigger_combo分組計算 win rate / payoff / expectancy / max drawdown
2. **ATR對照組**:同訊號、ATR門檻(2.5×/1.5×ATR14)平行模擬;兩組expectancy差距 < 0.5%(單筆)則正式版採FIXED
3. **Regime切片**:三種行情分段檢視勝率衰減幅度,任一combo在單一regime貢獻全部expectancy者標註「regime依賴」
4. **參數敏感度**:權重60/40掃描±15、共振視窗N掃描±5日、便宜門檻掃描±10分——expectancy對參數的曲面應平滑,尖峰即過擬合警訊
5. **防呆條款驗證**:比較「被防呆取消的訊號」若強行進場的假想績效,驗證取消條款是否真的濾掉壞訊號
6. **交易成本**:台股單邊0.1585%+賣出證交稅0.3%、美股假設0.05%滑價,全數計入(小市值標的滑價加倍)

## 4. 歷史勝率的誠實預估區間

基於同類「籌碼動能×估值過濾」策略在台美市場的公開研究與payoff 1.6的門檻結構,合理預期落點:

- **勝率**:42–52%(損益兩平線38.5%,超出即正expectancy)
- **單筆expectancy**:+0.5% 至 +2.0%(扣除成本後)
- **年化訊號數**:台股約80–150筆、美股約40–80筆(Form 4觸發較稀疏)

若回測結果顯著優於此區間(如勝率>60%),第一反應應是檢查lookahead與存活者偏差,而非慶祝。

## 5. 已知失效環境(誠實清單)

1. **系統性風險跳空日**:−5%停損被跳空穿越(GAP_SL),實際虧損大於模型;回測將2024/8/5式的急跌日單獨切出,報告GAP_SL佔比與實際vs理論虧損差
2. **投信季底作帳期**(3/6/9/12月最後兩週):連買訊號勝率系統性虛高,回測對此期間觸發的訊號單獨標記並比較
3. **市場快速輪動期**:13F背景因子在AI基建→應用層這類快速輪動中近乎無資訊量(45天延遲致命)
4. **流動性枯竭期**:日均量門檻在恐慌期會使universe急縮,候選清單可能連續數週為空——這是特性不是bug,但需在報告中說明
5. **估值指標的產業性失效**:橫斷面「後30%便宜」在整個產業被重估時(如2024 AI伺服器)會系統性錯過最強標的——便宜過濾的代價是放棄動能極端值,需明示此trade-off

## 6. 產出物

回測完成後產出:(1) 全期間績效總表(依expectancy排序、warn_flag標警)(2) regime切片報告 (3) 參數敏感度熱力圖 (4) 失效環境分析章節。全部進入週報系統的同一套 combo_performance 表,實盤數據累積後可直接比對「回測 vs live」的績效衰減。
